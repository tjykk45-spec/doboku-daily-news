# 目的

土木 Daily News 朝刊ダッシュボード（[index.html](../../../index.html)）に載せる1記事分のJSONを作る。

このスキルは**要約のみ**を担当する。HTML生成・差分判定・ファイル操作は**コード側**がやる（[AGENTS.md](../../../AGENTS.md) 参照）。

---

## 入力

呼び出し側から以下が渡される想定:

- `title_raw`: 記事タイトル
- `url`: 記事URL
- `body` または `lead_raw`: 記事本文（有料壁で取れない場合はリード文のみ）
- `published_at`: 公開日時
- `source_name`: 取得元の名前（例: `国交省 報道発表`）
- `section_hint`: コード側が判定した仮セクション（任意）

## 出力（厳守）

必ず以下のJSONで返す。`response_mime_type: application/json` 前提。**文章を装飾しない**。

> **スキーマ版**: v1.1（2026-04-26 更新。`is_in_scope` / `maturity` を追加）

```json
{
  "title": "string (90文字以内・原文の事実関係を変えない)",
  "url": "string (入力の url をそのまま)",
  "section": "国交省・規制・政策 | 技術・トレンド | 資材価格・コスト",
  "date": "YYYY-MM-DD (published_at から抽出)",
  "is_in_scope": "boolean (土木中心の本朝刊スコープに直接該当するか)",
  "maturity": "確定 | 提言 | 観測",
  "lead": "string (100文字前後・敬体で)",
  "bullets": ["string", "string", "string"],
  "tags": ["#国交省", "..."]
}
```

### 各フィールドのルール

- **title**: 原文の固有名詞・数値は改変しない。短縮時も**意味を変えない**。90文字以内。
- **section**: 値は以下3つだけ。該当なしなら最も近いものを選び、`bullets` の1つに「セクション判定に自信なし」と明記する。
  - `国交省・規制・政策`
  - `技術・トレンド`
  - `資材価格・コスト`
- **is_in_scope**: 本朝刊は「土木中心（国交省公共工事メイン）」が軸。以下に該当するなら `true`:
  - 土木工事・公共インフラ（道路・河川・橋梁・港湾・上下水道など）に関する発表
  - 建築・土木を横断するトピック（i-Construction、BIM/CIM、自動運転建機、DX、資材価格指数、働き方改革）
  - 公共工事全般の規制・政策（建設業法・入札・積算基準）
  
  「公共建築（営繕）」専用など**建築のみに閉じる**内容は `false`。`false` でも朝刊には載るが、コード側で「スコープ外参考」バッジを付ける想定。
- **maturity**: 内容の成熟度。
  - `確定`: 施策・基準として確定しており、施行日や適用範囲が明記されている
  - `提言`: 有識者・委員会・勉強会等の提言段階で、制度化はこれから
  - `観測`: 統計・市場動向・事例報告など、即施策化ではないが業界観測として重要
- **lead**: 1段落・敬体（です・ます）・100文字前後（±20文字まで）。**意見を混ぜない**。
- **bullets**: 読者（土木業界の管理職）が意思決定に使える観点を最大3つ。
  - 「いつまでに」「誰に」「何が」影響するか、を意識する。
  - `maturity: 提言` の場合は **「施策化の見込み」または「次に追うべき会議体」** を1つ入れると読者価値が上がる。
  - 想像での補足を入れない。書けないなら2つでも可（ただし最低1つ）。
#### 発表日フォールバック（コード側から渡される場合）
- `published_at` に `取得日代替` のフラグが付いている場合、
  `lead` の末尾に `（発表日不明のため取得日で代替）` を必ず入れる。

#### 本文薄い記事の扱い
- コード側から `body_length: N` が渡され、**300文字未満**の場合：
  - `bullets` は最大 **1件** に絞る
  - bullets の末尾に `（詳細は別添PDF参照）` を明記する
  - `lead` は取得できたリード文のみで構成し、推測で膨らませない

- **tags**: [AGENTS.md の許可タグリスト](../../../AGENTS.md#5-許可タグリスト) 内から選ぶ。外の値は**絶対に出さない**。

---

## 禁則

- 固有名詞の言い換え（例: 「国交省」→「国土交通省」の勝手な統一も禁止。原文に従う）。
- 数字の丸め（「約500億円」を「500億円」にしない）。
- 原文にない断定（「〜は確実」「〜が予想される」などの推測）。
- 本文が取れていないのに推測で要約を膨らませる。本文欠落時は `lead` に `(本文未取得のため見出しとリードのみ)` と明記。
- 同じ内容の bullet を複数出す。
- HTMLタグ・Markdown記号（`#`, `*`, `>`, `-` など）を `lead` `bullets` の中に混ぜる。
- 出力の前後に説明文や `json` のような囲い文字を付ける。**JSONだけ**を返す。

## 良い例 / 悪い例

### 良い例

入力（抜粋）:

> 国土交通省は令和8年度の公共工事積算基準改定案を公表した。技能者単価の上昇を一部反映するなど、設計積算の前提が変わる可能性がある。意見募集は11月30日まで。

出力:

```json
{
  "title": "国交省、令和8年度の公共工事積算基準改定案を公表 意見募集は11月30日まで",
  "url": "https://www.mlit.go.jp/report/press/sample.html",
  "section": "国交省・規制・政策",
  "date": "2026-04-24",
  "is_in_scope": true,
  "maturity": "確定",
  "lead": "国土交通省が令和8年度の公共工事積算基準改定案を公表しました。技能者単価などの前提が変わるため、設計積算に影響する可能性があります。意見募集は11月30日までです。",
  "bullets": [
    "設計積算の前提が一部見直される可能性があり、進行中案件の再試算が必要になることがある",
    "意見募集は11月30日までのため、社内の関係部門と読み合わせる時間を逆算する必要がある",
    "一次情報は国交省の報道発表ページを確認"
  ],
  "tags": ["#国交省", "#政策", "#規制"]
}
```

### 悪い例（やってはいけない）

- ` ```json ` で囲む
- `lead` に「これは非常に重要なニュースです」などの感想
- `tags` に `#朗報` `#管理職必見` など許可リスト外を入れる
- 本文未取得なのに勝手に「単価は約10%上昇する見込み」と推測を書く
