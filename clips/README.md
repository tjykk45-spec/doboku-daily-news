# 後追い・プロジェクト用クリップ

朝刊ページの **「後追い用にコピー」** でクリップボードに入る Markdown を、ここに貼り付けて蓄積します。

## おすすめの置き方

| やり方 | 向いている人 | メモ |
|--------|--------------|------|
| **このフォルダの `inbox.md`** | 個人・Git で履歴管理したい | 日付見出しの下に貼るだけ。後から `projects/` に振り分け。 |
| **スプレッドシート** | チーム共有・フィルタ・担当割り | 列例: `日付` `タイトル` `URL` `セクション` `要約` `プロジェクト名` `次アクション` `ステータス` |
| **`projects/*.md` 1ファイル1テーマ** | 特定案件だけ深掘りしたい | フォルダは `clips/projects/`。コピーMarkdownの `プロジェクト:` 行を実際のコード名にしてから貼る。 |

## 自動化フェーズでやること（メモ）

- 前日と同一URL・同一要約なら **本日の面からは省略**（「継続ウォッチ」欄に1行だけ、など）。
- ソースの `Last-Modified` やフィードの更新日時が変わったときだけ **再掲・差分** を出す、というルールにすると「毎日同じ話」を避けやすいです。

## SSOTと「自動で md に溜める」

- **時系列ログ**（例: `clips/log/2026-04.md`）… 追記だけのタイムライン。振り返り・検索の一次受け皿にする。
- **プロジェクトの正本**（例: `clips/projects/積算基準改定.md`）… テーマごとに1ファイル。中身は更新・整理し、古い記事の繰り返しは書かない。
- 同じ記事をログにも正本にもフルテキストで複製し続けると **無駄が増える** ので、ログは短く、詳細は正本に寄せると整理しやすい。
- ブラウザのHTMLからローカルへ **無言で追記**するには、別途スクリプトやCI（講義3の「処理する場所」）が必要。まずはコピー運用か、日次ジョブで `log/` へ1行appendから始めると安全。
